# 签证与入境材料核查清单

第 2 步的目标不是"提一句要不要签证"，而是产出一份用户拿着就能照做的准备方案。整理结果时按下面的结构走，最后写入 `visa_notes.md`。

## 1. 先判断属于哪一类

| 类型 | 典型特征 |
|---|---|
| 免签 | 持指定护照可直接入境，通常有最长停留天数限制 |
| 落地签 | 抵达口岸现场办理，通常需要预先准备照片、费用（现金/外币） |
| 电子签证 (e-Visa) | 出发前在线申请，审批后打印确认函或电子签 |
| 普通签证 | 需要向使领馆或签证中心递交材料，通常耗时较长 |
| 过境免签 | 仅适用于经停/转机，有停留时长和是否需要离开机场的限制 |

## 2. 信息源优先级

1. **目的地国家官方移民/边境管理局网站**（最权威）
2. **目的地国家驻用户所在国的使领馆官网**
3. **用户所在国外交部的出境提醒/领事服务页面**
4. 第三方签证服务机构（如 VFS Global、签证中心）官网——可以用来确认材料清单和预约流程，但最终条件以前三者为准
5. 旅游攻略/社交媒体内容——只作为"听说过这种情况"的线索，不能作为最终依据，尤其是免签天数、材料清单这类硬信息

至少交叉验证两个独立信息源，如果说法不一致，明确告诉用户存在分歧。

## 3. 免签 / 落地签 / 电子签需要确认的信息

- 最长可停留天数
- 入境时可能被要求出示的辅助材料：往返机票/离境机票、酒店预订确认单、资金证明、（如适用）邀请函
- 落地签/电子签的费用、支付方式（是否只收现金/特定货币）
- 电子签的申请网址、审批通常需要多久、是否需要提前打印纸质确认件

## 4. 普通签证需要确认的完整清单

- **材料清单**：护照原件（及剩余有效期要求）、近期免冠照片（规格）、签证申请表、行程单、酒店预订证明、资金证明（流水/存款证明）、在职证明或在学证明、旅行保险、（如适用）邀请函
- **申请渠道**：使领馆直接受理还是必须通过指定签证中心
- **费用**：签证费 + 签证中心服务费（如有）
- **办理时长**：官方标注的工作日数，并建议预留缓冲（旺季/材料补交都会拖慢）
- **有效期与可入境次数**：单次/多次，有效期多长
- **建议办理时间点**：用出发日期倒推，明确告诉用户"建议从出发前第 X 周开始准备材料"

## 5. 容易被忽略但同样重要的细节

- 护照剩余有效期要求（很多国家要求入境时护照至少还有 6 个月有效期）
- 护照空白页数量要求
- 强制性的电子入境登记/旅行授权（类似 ESTA、eTA、K-ETA 这类，免签也可能需要额外做这一步）
- 健康相关要求：是否需要特定疫苗接种证明（如黄热病疫苗）、是否有健康申报系统
- 海关/物品携带限制（如有特别突出的，可以提一句，不必穷举）

## 6. 写入 `visa_notes.md` 的建议结构

```markdown
## 签证与入境材料

**结论**：[免签/落地签/电子签/普通签证]，最长停留 X 天 / 需提前 X 周办理

### 准备材料清单
- [ ] ...
- [ ] ...

### 申请渠道与费用
...

### 时间规划
建议从出发前第 X 周开始准备，预计审理时长 X 个工作日。

### 其他注意事项
（护照有效期、电子入境登记、健康要求等）

### 信息来源
- [来源1名称及链接]
- [来源2名称及链接]

> 签证政策可能随时调整，以上信息基于查询时的公开信源整理，请在临近出发前以官方渠道公布的最新信息为准。
```

把这份 `visa_notes.md` 转写进最终网页时（第10步），"申请渠道与费用"和"时间规划"这类**有先后顺序的办理流程**，在 HTML 里用 `<ol class="steps"><li>...</li></ol>`（模板自带的圆形编号步骤样式），不要写成一段夹杂"1. 2. 3."的纯文字；"准备材料清单"这类并列条目用普通 `<ul><li>` 即可。具体写法参考 `html_content_guide.md`。

## 7. 务必向用户强调

- 签证结果未确定前，机票酒店尽量选可改期/可退的选项。
- 这份清单是规划参考，不构成法律或移民建议；遇到复杂情况（曾被拒签、双重国籍等）建议直接咨询使领馆或专业签证机构。
