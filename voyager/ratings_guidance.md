# 地点评分与评价查找指南

第3/6/8步搜索住宿、餐饮、景点时，顺手把评分、评价数量和一句话点评摘要记下来。这些信息是用户决策的重要参考，也会显示在最终网页的时间轴和“地点图鉴”里。

核心原则：**评分/评论数据源按地点所在地区选择，不跟在线地图绑定**。页面内在线地图固定用 OSM；地点图鉴里的评分、评论摘要和评价数量必须优先使用该地区覆盖最好的平台。

## 中国内地与中国内地以外分开处理（强制）

- **目的地在中国内地**：默认用国内软件和平台获取评分/评价，例如携程、飞猪、去哪儿、美团、大众点评、高德地图、马蜂窝、小红书等。餐饮优先大众点评/美团/高德，住宿优先携程/飞猪/去哪儿，景点优先携程/大众点评/马蜂窝/高德。
- **目的地在中国内地以外（含港澳台和海外国家/地区）**：默认用境外常用来源获取评分/评价，例如 Google Maps、Tripadvisor、Booking.com、Agoda、Yelp（适用地区）、官方票务/预订平台等。

使用境外来源时，只需要提醒用户一句：**“接下来会用境外来源核对部分评分、评论或图片。”**提醒到这里即可。

## 英文名 + 本地语言名一起搜

查评分和评价时，不要只用中文译名。优先尝试：

1. 英文名，例如 `Blue Lagoon Iceland`。
2. 当地语言名，例如 `Bláa lónið`。
3. 完整地址或城市/国家组合，例如 `Norðurljósavegur 9, 240 Grindavík, Iceland`。
4. 官方名称、别名或票务平台常用名称。

建议把这些名称写入 `location_en`、`location_local`、`address`、`search_aliases`，这样后续查图和 OSM 坐标也能复用。

## fallback 规则：首选搜不到就试其它服务

1. 先按地区选择主数据源：
   - 中国内地：国内平台优先。
   - 中国内地以外：Google Maps / Tripadvisor / Booking / Agoda / Yelp 等境外平台优先。
2. 首选来源没有评分或评价数量时，尝试同地区的其他主流平台。
3. 仍然没有时，可以尝试另一类来源作为补充：
   - 内地地点：可补查境外平台，但只作为辅助参考。
   - 境外地点：可补查携程/飞猪/马蜂窝等中文平台，但只能当补充，不作为默认主来源。
4. 所有来源都查不到时，字段留空，不要编造。

不管用哪个平台，都必须在 `rating_source` 里如实标注来源。不要把不同平台的分数混成一个“综合评分”，除非你明确说明计算方式；默认只记录最可信、最贴近该场景的一项评分。

## 从哪查

- **住宿**：
  - 中国内地：携程、飞猪、去哪儿、同程等预订平台。
  - 中国内地以外：Booking.com、Agoda、Google Maps、Tripadvisor、Expedia 等。
- **餐饮**：
  - 中国内地：大众点评、美团、高德地图、携程美食。
  - 中国内地以外：Google Maps、Tripadvisor、Yelp（当地适用时）、本地点评平台。
- **景点/活动**：
  - 中国内地：携程、马蜂窝、大众点评、高德地图、景区官方购票渠道。
  - 中国内地以外：Google Maps、Tripadvisor、GetYourGuide/Klook/Viator 等活动平台、官方票务平台。

小红书可以用来补充真实体验、近期状态、排队情况和小众发现，但价格、营业时间、预约规则这类硬信息要优先以官方或主流预订/点评平台核实。

## 记录格式

写入 `trip_data.json` 对应 item 的字段：

- `rating`：分数本身，尽量统一换算成 5 分制方便对比；如果平台分制特殊或不适合换算，就在 `rating_source` 里注明分制，例如 `"Booking.com(满分10分)"`。
- `rating_source`：评分来源平台，例如 `"大众点评"`、`"高德地图"`、`"Google Maps"`、`"Tripadvisor"`。
- `rating_count`：评价数量，没有就留空。
- `review_summary`：一句话点评摘要，必须用自己的话转述网友评价的共性要点，绝不能逐字摘抄某条评论原文。

## 找不到评分/点评怎么办

找不到很正常，尤其是小众地点、刚开业店铺、临时活动，或境外地点在国内平台没有覆盖。直接留空，不要编一个看起来合理的分数或点评。

例如 Blue Lagoon, Grindavík, Iceland 这类境外 POI，可以直接用英文名、当地语言名、完整地址或坐标去境外来源查，不要把国内平台查不到当成“地点不存在”。

## 在最终网页里怎么呈现

只要 `trip_data.json` 里的 item 有 `rating` / `review_summary` / `photo_url`，`itinerary_html_builder.py` 会自动展示：(1) 逐日时间轴里的评分徽标；(2) “地点图鉴”里的照片、评分和点评摘要。第12步直接复用脚本输出，不要手写。

## 跟用户呈现选项时也要带上评分

第3/6/8步给用户看 A/B/C 几个选项时，评分应和价格、时长、位置等信息并列展示。找不到评分的选项可以不写评分栏，但不能编。
