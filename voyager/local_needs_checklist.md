# 本地特殊需求核查清单

第 5 步不是可有可无的客套提问——很多"行程本身没问题，但卡在当地办不成"的情况都出在这一类需求上。下面以**租车自驾**为完整范例，并列出其他几类常见需求供按同样思路处理。**租车自驾这个需求现在有两个触发入口**：第4题直接选了"自驾租车"，或者第4题选了别的、但在第5步这里额外问出来用户确实需要租车——不管从哪个入口进来，后续的驾照核实和准备清单流程是一样的。

## 租车自驾（完整范例）

### 驾照与资质
- **第5步已经先问清楚了用户驾照的颁发国家/地区（`trip.license_country`）和类型（`trip.license_type`）**，研究这部分时要针对用户的具体情况给出明确结论（比如"中国大陆C1驾照在日本不能直接使用，需要办理符合日内瓦/维也纳公约标准的驾照翻译认证件"），不要写成"建议核实当地政策"这种谁都适用的空话。
- 用户实际驾照能否在目的地直接使用，还是必须搭配国际驾驶许可证（IDP）/驾照公证翻译件——**这条结论直接影响能不能取到车、是否合法驾驶，建议至少两个信息来源互相印证**，如果来源之间有分歧要如实告诉用户。
- 最低驾龄要求（很多公司要求持照满 1-2 年）
- 最低年龄要求，以及"年轻驾驶员附加费"（通常 21-25 岁以下会有）

### 交通规则与路况
- 靠左还是靠右行驶（如果和用户本国不同，要明确提醒，这是安全问题）
- 限速标准、是否对外国驾照有特殊限制路段
- 强制随车装备（三角警示牌、反光背心、灭火器等，部分国家强制要求）
- 收费路段/桥梁的支付方式（人工收费、电子标签 ETC 类、车牌识别自动扣费）

### 租车本身的要求
- 主驾驶人名下需要有信用卡（不是借记卡）且额度足够覆盖押金
- 基本保险 vs 全险/超级补充保险（CDW/SCDW）的覆盖范围差异，建议用户根据风险承受能力选择
- 取还车的油量政策（常见"满油取车满油还车"，否则会被收取较高的代加油费）
- 异地还车是否允许、是否收取额外费用

### 当地用车实际情况
- 目的地市中心是否难停车/停车费高，是否建议"市区用当地交通、出城再用车"
- 旺季/特定路段是否容易堵车，是否有限行政策

### 费用与渠道
- 搜索 2-3 个可参考的渠道（国际连锁租车公司 / 本地平台），给出预估日租金区间
- 把预估费用按 `category: "租车/装备"` 写入 `trip_data.json`

## 其他常见本地特殊需求（按同样思路处理：要不要提前预约/资质要求/费用/渠道）

- **长途包车 / 司机服务**：是否需要提前多久预约、是否包含过路费油费、司机是否讲用户语言
- **滑雪/潜水等装备租赁**：是否需要资质证书（如潜水需要 OW 证书）、装备押金、是否需要提前体检或培训
- **本地 SIM 卡 / eSIM**：是否需要护照实名登记、机场/市区购买的价格差异、eSIM 是否提前线上购买更划算
- **货币兑换**：目的地是否容易使用信用卡/移动支付，建议携带多少现金、在哪里兑换汇率更划算
- **宠物友好安排**（如果用户带宠物出行）：交通工具是否允许、住宿是否接受、当地是否有相关入境检疫要求
- **无障碍需求**（如果用户提到行动不便或带老人小孩）：住宿/景点/交通的无障碍设施情况

## 写入 `local_needs_notes.md` 的建议结构

每一类需求一个小节：

```markdown
## 租车自驾

### 驾照能不能用
（针对用户实际驾照情况给出的具体结论，例如："你持有的中国大陆C1驾照在新西兰可以直接使用，无需办理IDP，但建议随身携带驾照的英文公证翻译件，部分租车公司会要求查验。"）

### 准备清单
- [ ] 确认信用卡额度足够支付押金
- [ ] 准备驾照英文翻译件（如适用）
- [ ] ...

### 预估费用
日租金约 ¥XXX-XXX，已计入预算"租车/装备"类别。

### 推荐渠道
- [渠道1]
- [渠道2]

### 特别提醒
（靠左行驶/强制装备/还车油量政策等）
```

把这份 `local_needs_notes.md` 转写进最终网页时（第11步），"准备清单"如果是**有先后顺序的步骤**（比如"先办IDP、再租车、最后买保险"），在 HTML 里用 `<ol class="steps"><li>...</li></ol>`（模板自带的圆形编号步骤样式）；如果只是一堆并列要点、没有先后关系，用普通 `<ul><li>` 就好，不必强行编号。具体写法参考 `html_content_guide.md`。

如果用户没有任何额外本地需求，直接写：

```markdown
本次行程无额外本地特殊需求。
```

不要省略这个文件本身，即使内容只有这一句话，第 10 步生成最终文档时也需要读取它来填充"本地特殊需求"板块。
